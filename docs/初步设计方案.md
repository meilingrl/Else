## 一、项目名称与开发愿景

### 1.1 名称

- **中文**：未择命运
- **英文**：**Else**  
- **含义**：呼应代码中的 `if / else`——未进入的 `if` 分支沉入 `else`；被放弃的世界线堆积为黑暗与难度。极简四字，利于 Steam 等商店视觉呈现。

### 1.2 愿景（一句话）

打造一款 **视觉极简、操作上限高、叙事与机制咬合** 的 **2D 多人合作动作 Roguelike**：用最少的美术语言，承载「未择命运」「统一时间」「维度跳跃」的高概念，并允许单人先行、联机可扩展。

### 1.3 核心标签

极简 2D（像素）｜联机 PvE（远期）｜一滴血开局｜类魂式博弈（弹反 / 闪避 / 构筑）｜定向盲跳 Roguelike｜时间—仇恨—难度闭环

### 1.4 情感与视听基调

- **情感**：极度紧张（零常规血量）、高成就感（化解危机 + 构筑成型）、合作羁绊或压力。  
- **视觉**：强对比色块 / 几何 / 清晰预警符号；极简服务于「判读攻击」而非省成本唯一目的。  
- **听觉**：预警与节奏（蓄力、打铁、滴答）参与玩法；仇恨升高时 BGM 与低频压迫感递进。  
- **搞怪**：允许怪诞 Boss 与反差命名，但需 **有设计的用心**（非纯玩梗）：打击感、顿帧、 glitch 与叙事一致。

## 二、世界观与叙事（玩法逻辑闭合）

### 2.1 维度模型（2D 像素下的「升维」叙事）

- 世界由无数 **2D 平面** 叠成，类似书页 / 扑克牌式纵深（Z 轴为世界线，屏幕内用透视、洗牌式转场、视差表现）。  
- **时间线**：全宇宙 **统一、单向、不可回退**，是公平且可读的「全局时钟」。  
- **世界线**：玩家在平面间 **跳跃** 改变「处于哪一层现实」，不改变时间倒流。  
- **联机包装**：队友可叙述为「另一世界线里做了相近选择的你」— 时间线短暂共鸣。

### 2.2 命运、仇恨与难度（Ludonarrative）

- 每次推进 / 跳跃，**未被选择的可能性** 被黑暗吞噬，转化为全局 **仇恨（Hatred）** 或类似度量。  
- **时间与仇恨绑定**：拖得越久、逛得越多，黑暗蔓延越深 —— 用世界观解释「难度随时间上升」，避免纯数值条无叙事支撑。  
- **表现层建议**：色调随仇恨渐变、深处噪点 / glitch、BGM 混入心跳与秒针；背景视差中偶现他线崩塌剪影。  
- **环境威胁**：高维「天灾」、黑暗污染地面、地面崩坏等，使小怪阶段也感受终局压力。

### 2.3 终局目标与「凡人弑神」数值梗

- 玩家终须回到 **主世界线** 面对最终 Boss；可早归主线或长期边缘游荡（无限 Rogue），后者拉高仇恨与 Boss 强度。  
- **开局可见的极端数值**：例如玩家基础攻击力 **5**、Boss **20000 HP**。  
  - **极限挑战**：一滴血 + 若弹反链可维系，理论上存在「超低攻磨死」的速通 / 挑战向可能。  
  - **常规模型**：通过构筑把 DPS、机制（真伤、引爆 DoT 等）拉到人类可接受通关时长。
- Boss 随 **时间与仇恨** 强化（形态、频率、弹幕形态、阶段等），防止「神装 + 短局」无代价碾压。

## 三、核心玩法机制

### 3.1 生存与资源

- **开局一滴血（1 HP）**：任何未化解的伤害 → 死亡（局内规则以具体模式为准，见 §8 待决项）。  
- **有限闪避**：次数受限，用于走位与非锁定威胁；恢复规则 **待决**（时间回复 vs 进攻/弹反充能）。  
- **弹反**：  
  - **精准成功**：无 CD（或短内置节奏），可形成「弹反流」。  
  - **失误**：长 CD，在一滴血框架下极易形成致命空窗。
- **定位修正**：弹反是 **高上限手段之一**，不是唯一解；需工程、DoT、无人机、护盾等 **宽下限** 构筑，以降低联机与上手门槛，并允许借鉴《元气骑士》《雨中冒险》类「本地判定优先 + 设计掩盖延迟」的思路。

### 3.2 Boss 战为核心的三角攻击矩阵


| 类型           | 玩家应对                       | 与构筑关系                          |
| ------------ | -------------------------- | ------------------------------ |
| **单体锁定**     | 强预警；原则上 **弹反** 化解（难以纯走位逃课） | 弹反收益、团队标记可读性                   |
| **弹幕**       | 走位 + **墙体** 等掩体            | 工程流；墙体吃耐久                      |
| **延迟范围 AoE** | 走位 / 拆阵                    | **无视或击穿掩体**逻辑，墙体与玩家可被一并打击，防止龟缩 |


**碰撞层级思路（实现时）**：区分 Player、玩家造物（墙 / 无人机）、Boss 弹幕实体、Boss AoE 触发区等；弹幕撞墙扣墙耐久；AoE 为地面触发器，按规则伤害范围内单位与建筑。

**联机可读性**：单体锁定时 **明确 target_id** 同步；极简下用红线、倒计时音效等强标记 **谁必须弹反**。

### 3.3 关卡流转：Boss 后的空间裂缝

- 击败 Boss 后出现 **空间裂缝**（极简 glitch / 黑色非几何裂缝等演出）。  
- **选择**：  
  - **留在当前平面**：继续探索当前维度节点；**时间仍在走**，仇恨上升，可能触发平面崩坏、生存压力。  
  - **跃迁**：跳入裂缝，选 **向上 / 向下**（定向）。  
    - **向下**：偏贪婪 / 高危 / 高稀有奖励倾向。  
    - **向上**：偏秩序 / 休整 / 较低污染。
  - **盲跳幅度**：方向可控，**跨越多少层随机**，可能 overshoot 进超高难节点。
- **种子同步**：服务器（或主机）发种子，各端本地生成一致地图，减少同步流量（远期联机）。

### 3.4 构筑流派（系统需可扩展）

以下为对话中已列举的方向，实现上适合 **标签 / 事件总线（如 OnParrySuccess）+ 数据驱动**，避免硬编码：

- 攻速暴击 / 攻速触发  
- 一击秒杀（资源与大硬直窗口）  
- DoT 叠层（流血、灼烧）；病毒扩散（可结合弹反引爆层数）  
- 建造工程（墙挡弹幕；资源上限；AoE 拆阵）  
- 无人机（仇恨规则需设计：是否吸引 Boss）  
- 叠护盾（脱战恢复；破盾代价）  
- 弹反盾辅（多人：群体短无敌或护盾，需广播）  
- 移速流  
- 「时间机器原理」类：**改规则**（如跳跃可控性、窥视节点危险度）— 要求关卡生成器可读玩家 Buff  
- 其他：Proc 减伤、清屏、以攻代守等可吸收《雨中冒险》式爽感

### 3.5 Boss AI 与后期弹幕

- Boss 需多阶段行为（追击、弹幕、召唤、疲劳窗口等），否则单一 AI 无法承载多流派。  
- 后期弹幕可进化：曲线、弹射、穿甲弹等，压缩工程流安全区。

## 四、多人合作（设计原则 + 技术方向）

### 4.1 模式

- **主目标**：多人 **PvE 合作 Boss**（非 PvP）。  
- **判定哲学**：借鉴《雨中冒险》「本地爽感优先」、**反转**为 **防守方客户端权威**：被锁定玩家本地判定弹反 / 闪避成功，服务器信任并下发 Boss 硬直等状态；旁观端允许轻微视觉修正。  
- **作弊**：买断制 PvE 可权衡；若商业化的竞技榜需另议。

### 4.2 网络（规划层）

- **协议倾向**：动作同步用 **UDP**（或可靠 UDP 封装），避免 TCP 队头阻塞。  
- **拓扑**：Host-Client / P2P + 中继（NAT 失败时）；EOS / Steamworks / 云厂商中继等可后续接入。  
- **平滑**：插值、航位推测、流量控制，缓解 Boss / 弹幕密集时拥塞。  
- **与《元气骑士》对比**：其主机算怪 + 自瞄与高容错；本作一滴血更苛刻，故 **本地防守优先** 更关键。

## 五、软件架构原则（单机先行，联机可插拔）

### 5.1 为何先单机

- 项目整体难度评估 **偏高**（约 8/10 量级）：最大风险在 **网络 + 判定一致性**。  
- **策略**：先 **单机验证手感与帧判定**，底层按联机友好方式搭建，避免日后推倒重来。

### 5.2 必须遵守的三条工程原则

1. **输入与逻辑解耦（Command Pattern）**
  输入 → 生成指令对象 → 缓冲 / 执行；联机时在「生成」与「执行」之间插入网络层，**战斗逻辑尽量少改**。
2. **固定逻辑步长（Fixed Timestep）**
  逻辑帧（碰撞、状态机、弹反窗）与渲染帧分离；逻辑层不直接调用渲染 API，用事件驱动表现。
3. **状态可序列化（快照 / 回放）**
  便于调试、死亡回放、未来同步与冲突解决；与「时间线」主题可玩法化。

### 5.3 进阶选项（非第一周必做）

- **定点数 / 确定性物理**：多平台严格一致时考虑；初学阶段可用引擎 2D 触发器 + 约束规范过渡。  
- **Rollback**：格斗级方案，成本高；PvE + 防守方权威可大幅降低刚需。  
- **ECS / FSM / 帧数据表**：Roguelike 实体多、动作游戏需 **前摇 / 判定帧 / 后摇** 数据驱动，适合中期模块化。

### 5.4 性能

- **对象池**：弹幕、碎片、临时造物。  
- **空间划分**：四叉树等，控制碰撞对数。  
- **目标帧率**：移动 60fps；PC 可支持更高刷新，逻辑步长仍固定。

## 六、技术栈建议（单人、零基础友好路径）

对话中形成 **两套语境** 的结论需并存：


| 维度  | 理想 / 长期               | 单人起步（当前推荐）                                                   |
| --- | --------------------- | ------------------------------------------------------------ |
| 引擎  | Godot 4.x **或** Unity | **Unity（2022 LTS / Unity 6）** — 教程、AI 辅助、Asset Store、跨平台案例最多 |
| 语言  | C# / GDScript         | **C#**                                                       |
| 物理  | 长期可考虑定点数 + 自研 AABB    | **Raycast2D / Trigger2D**，快速验证                               |
| 网络  | UDP + 防守方权威           | **不做**，但保留 Command 入口                                        |
| 工具链 | 自定                    | **Cursor** 辅助编码；版本管理习惯尽早建立                                   |


**与《元气骑士》对齐的参考栈**：Unity、UDP、P2P/主机、对象池、2D 触发器与射线、可选 PlayFab / Addressables 等（远期）。

**Godot vs Unity 摘要**：Godot 更轻、2D 工作流纯粹；Unity 生态与商业插件、主机移植经验更厚。一人入门优先 **完成度**，故推荐 Unity。

## 七、美术与音乐（有心、可搞怪、可用 AI 辅助）

### 7.1 美术

- **方向**：极简霓虹像素 + **后处理**（Bloom、色差、震屏、顿帧）抬升质感。  
- **流程**：MJ / SD 概念与情绪板 → **Aseprite** 规范像素网格；避免直接沿用带「伪像素」的 AI 出图。  
- **搞怪**：几何体 + 反差角色（如对话中的咸鱼巨物等）需与 **可读预警** 平衡。

### 7.2 音频

- **动态音乐**：Suno / Udio 分段素材 + **FMOD / Wwise** 按仇恨 / 时间参数切换层。  
- **关键 SFX**：弹反打铁、死亡等建议素材库精修 + 混音，保证短促利落、0 延迟感。

### 7.3 界面彩蛋（可选）

- 标题 `**else_`** 闪烁光标；开始游戏时可玩 `**if (you.play)`** 切入 — 强化命名记忆点。

## 八、待决设计项（需你拍板）

1. **团灭规则**：一人即死重开（硬核）vs 灵魂 / 倒地可复活（类魂救援）等。
2. **闪避恢复**：随时间回 vs 仅攻击命中 / 弹反成功充能。
3. **单体锁定仇恨**：随机 vs 距最远 / 正在建造等战术规则。
4. **防作弊程度**：纯 PvE 信任客户端的边界。

## 九、分阶段开发路线（建议）

1. **白盒单机 MVP**：方块玩家 + 简单 Boss；1 HP；有限闪避；弹反窗 + 单体锁定一类攻击；Command 输入；顿帧与基础 Juice。
2. **Boss 三角验证**：弹幕 + 墙耐久；AoE 拆墙；碰撞分层。
3. **Roguelike 骨架**：种子、节点、裂缝上下跳、仇恨随时间爬升（可先简化公式）。
4. **构筑与数据驱动**：遗物、事件、数种流派原型。
5. **联机攻坚**：假延迟测试；防守方权威；主机 + 中继。
6. **包装**：叙事演出、多语言、商店素材。

## 十、风险与难度评估（摘要）

- **最高风险**：联机同步 + 一滴血下的 **手感与公平感**；需用 **构筑多样性 + 判定分层** 缓冲。  
- **高复杂度**：极端数值下的 **成长曲线** 与 Boss 时间缩放；帧数据与 FSM。  
- **中等**：Roguelike 状态、多世界线存档结构、事件系统。  
- **相对可控**：极简渲染管线、AI 辅助美术音乐管线。
