# 《Else》里程碑与任务分解（单人开发版）

## 1. 目标

把“大方案”拆成可执行的小步任务，确保单人开发可持续推进。

---

## 2. 里程碑定义

## M1：项目底座（1-2 周）

- Unity 工程初始化与目录结构落地
- 输入 -> Command -> 逻辑执行通路打通
- 主菜单、游戏场景、失败重开闭环

**出口条件**：可稳定进入游戏并执行基础动作。

## M2：战斗白盒（2-3 周）

- 玩家基础战斗动作可用
- 一滴血规则生效
- Boss 原型（至少单体锁定 + 一种区域威胁）
- 基础 hit-stop 与反馈可见

**出口条件**：可进行可重复的白盒对战测试。

## M3：玩法闭环 MVP Alpha（2-4 周）

- Boss 三类攻击完整接入
- 裂缝跳层与基本节点流转可跑通
- 仇恨随时间变化并能影响体验
- 至少 3 条构筑雏形可玩

**出口条件**：可完整跑完一局循环。

## M4：MVP Beta（2-3 周）

- 核心体验打磨（可读性、手感、反馈）
- 主要性能问题收敛（对象池与加载稳定）
- 基础美术/音频风格统一

**出口条件**：可进行外部小范围试玩。

## M5：MVP 发布候选（1-2 周）

- 修复阻断级与高优先级问题
- 完成最小发布资料（截图、说明、版本记录）

**出口条件**：满足 `docs/02-MVP范围与验收标准.md` 中 DoD。

---

## 3. 任务分解模板（每周）

- 本周目标（1-2 条）
- 关键任务（最多 5 条）
- 风险点（最多 3 条）
- 验收方式（如何确认完成）
- 下周衔接（1 段）

---

## 4. 优先级规则

- 先保证“能玩完一局”，再做“更好看”
- 先解决阻断问题，再做体验增强
- 先做可复用基础设施，再做一次性内容

---

## 5. 风险与预案

- **风险**：任务超量导致延期
  - **预案**：每周砍掉低优先级内容，保主循环
- **风险**：技术卡点（单人）
  - **预案**：先降级方案，保留扩展接口
- **风险**：风格发散
  - **预案**：以 `docs/12-美术风格指南.md` 和 `docs/13-音频设计文档.md` 为唯一标准
